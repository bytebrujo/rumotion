Picus can be installed into existing projects, such as [Next.JS](https://nextjs.org/), [React Router](https://reactrouter.com/), [Vite](https://vitejs.dev/guide/) or [Create React App](https://create-react-app.dev/), as well as server-only projects that run on Node.JS. Get started by adding the following packages:

<Installation pkg="picus @picus/cli" />

- If you'd like to embed a Picus video in your existing React app, install [`@picus/player`](/docs/player/installation) as well.
- If you'd like to render a video using the Node.JS APIs, install [`@picus/renderer`](/docs/renderer) as well.
- If you'd like to trigger a render on Picus Lambda, install [`@picus/lambda`](/docs/lambda/setup) as well.

## Setting up the folder structure

Create a new folder for the Picus files. It can be anywhere and assume any name, in this example we name it `picus` and put it in the root of our project. Inside the folder you created, create 3 files:

```tsx twoslash title="picus/Composition.tsx"
export const MyComposition = () => {
  return null;
};
```

```tsx twoslash title="picus/Root.tsx"
// @filename: Composition.tsx
export const MyComposition: React.FC = () => {
  return null;
};
// @filename: Root.tsx
// ---cut---
import React from 'react';
import {Composition} from 'picus';
import {MyComposition} from './Composition';

export const PicusRoot: React.FC = () => {
  return (
    <>
      <Composition
        id="Empty"
        component={MyComposition}
        durationInFrames={60}
        fps={30}
        width={1280}
        height={720}
      />
    </>
  );
};
```

```ts twoslash title="picus/index.ts"
// @filename: Composition.tsx
export const MyComposition: React.FC = () => {
  return null;
};
// @filename: Root.tsx
import React from "react";
import { Composition } from "picus";
import { MyComposition } from "./Composition";

export const PicusRoot: React.FC = () => {
  return (
    <>
      <Composition
        id="MyComp"
        component={MyComposition}
        durationInFrames={60}
        fps={30}
        width={1280}
        height={720}
      />
    </>
  );
};
// @filename: index.ts
// ---cut---
import { registerRoot } from "picus";
import { PicusRoot } from "./Root";

registerRoot(PicusRoot);
```

The file that calls [`registerRoot()`](/docs/register-root) is now your Picus [**entry point**](/docs/terminology/entry-point).

:::note
Watch out for import aliases in your `tsconfig.json` that will resolve `import {...} from "picus"` to the `picus` folder. We recommend to not use `paths` without a prefix.
:::

## Starting the Studio

Start the Picus Studio using the following command:

```
npx picus studio picus/index.ts
```

Replace `picus/index.ts` with your entrypoint if necessary.

## Render a video

Render our a video using

```
npx picus render picus/index.ts MyComp out.mp4
```

Replace the entrypoint, composition name and output filename with the values that correspond to your usecase.

## Install the ESLint Plugin

Picus has an ESLint plugin that warns about improper usage of Picus APIs. To add it to your existing project, install it:

```bash
pnpm i -D @picus/eslint-plugin
```

  </TabItem>

This snippet will enable the recommended rules only for the Picus files:

```json title=".eslintrc"
{
  "plugins": ["@picus"],
  "overrides": [
    {
      "files": ["picus/*.{ts,tsx}"],
      "extends": ["plugin:@picus/recommended"]
    }
  ]
}
```
