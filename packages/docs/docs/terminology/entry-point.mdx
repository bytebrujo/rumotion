The entry point is the file where the [Picus CLI](/docs/cli) and [`@picus/renderer`](/docs/renderer) APIs will look for a Picus project.

- By default in most templates, it is `src/index.ts`.
- In older projects, it may have an `.tsx` extension instead of `.ts`.
- The entry point can be passed to the render command, for example: `npx picus render src/index.ts`.
- The entry point should call [`registerRoot()`](/docs/register-root).
- If you render a video using [`npx picus render`](/docs/cli/render), the entry point is printed in grey.
- You can customize the default entry point in the config file using [`Config.setEntryPoint()`](/docs/config#setentrypoint).

## Which entry point is being used?

If you call [`npx picus render`](/docs/cli/render), the entry point and the reason why it was chosen will be printed. The algorithm is as follows:

<Step>1</Step> If the entry point is directly passed, for example <code>npx picus render src/index.ts</code>, use that.<br/>
<Step>2</Step> Otherwise, if a path was specified in the <a href="/docs/config">Configuration file</a> using <a href="/docs/config#setentrypoint"><code>Config.setEntryPoint()</code></a>, use that. <br/>
<Step>3</Step> Go through the following common paths and if one exists, use that:

<ul>
<li><code>src/index.ts</code></li>
<li><code>src/index.tsx</code></li>
<li><code>src/index.js</code></li>
<li><code>src/index.mjs</code></li>
<li><code>picus/index.tsx</code></li>
<li><code>picus/index.ts</code></li>
<li><code>picus/index.js</code></li>
<li><code>picus/index.mjs</code></li>
<li><code>src/picus/index.tsx</code></li>
<li><code>src/picus/index.ts</code></li>
<li><code>src/picus/index.js</code></li>
<li><code>src/picus/index.mjs</code></li>
</ul>

## In the Player

In the [Picus Player](/docs/terminology/player), there is no concept of an entry point.  
You directly pass a React component and metadata to the [`<Player>`](/docs/player/player).
