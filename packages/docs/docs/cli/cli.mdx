## How to use

You can run the CLI by installing `@picus/cli` and running:

- `npx picus` inside a npm project
- `yarn picus` inside a Yarn project
- `pnpm exec picus` inside a pnpm project.
- `bunx picus` inside a Bun project

For brevity, in the documentation we always say `npx picus`.

Inside an npm script, you don't need the `npx` prefix:

```json title="package.json"
{
  "scripts": {
    "render": "picus render"
  }
}
```

### Using Bun<AvailableFrom v="4.0.118" />

By default, the `npx picus` command is being executed using Node.  
Even `bunx picus` is using Node, unless you add the `--bun` flag.  
To use Bun, replace `picus` with `picusb`.

```json title="package.json"
{
  "scripts": {
    "render": "picusb render"
  }
}
```

### Using Deno<AvailableFrom v="4.0.227" />

Deno is not supported by Picus.  
If you like to experiment nonetheless, use `npx picusd` to run the Deno version of the CLI.

```json title="package.json"
{
  "scripts": {
    "render": "picusd render"
  }
}
```
