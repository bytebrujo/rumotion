Adds one or more Picus packages to your project with the same version as your other Picus packages.

Also supports adding `zod` and `mediabunny` with the correct version for your Picus installation.

```
npx picus add <package-name...>
```

## Examples

Add a single package:

```bash
npx picus add @picus/transitions
```

Add multiple packages at once:

```bash
npx picus add @picus/transitions @picus/three @picus/lottie
```

This command will:

1. Verify that all package names are valid Picus packages (or supported extra packages: `zod`, `mediabunny`)
2. Check which packages are already installed (and skip them)
3. Detect the version of your currently installed Picus packages
4. Install the specified packages with the matching version

## Package manager support

`npm`, `yarn` and `pnpm` are all supported.

## Additional arguments

Any additional arguments you pass to this command will be forwarded as flags to the package manager, before the package name.

## Use case

This command is useful when you want to add new Picus packages to your project and ensure they're installed with the same version as your other Picus packages. This helps avoid version mismatches that could cause compatibility issues.

For example, if you have `picus@4.0.100` installed and you want to add `@picus/transitions`, running `npx picus add @picus/transitions` will install `@picus/transitions@4.0.100`.

When adding multiple packages, the command will install all of them in a single operation with the same version number.
