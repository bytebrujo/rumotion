It is important that all Picus packages you install (`picus`, `@picus/player`, `@picus/cli`, etc.) have the same version installed. If the versions differ, the packages might not be able to correctly communicate with each other which can introduce subtle bugs or even complete breakage.

## `^` character can be confusing

The following `package.json` does **not** ensure that all Picus versions are the same:

```json title="package.json"
{
  "dependencies": {
    "picus": "^2.6.12",
    "@picus/player": "^2.6.12",
    "@picus/gif": "^2.6.12"
  }
}
```

however the following manifest does:

```json title="package.json"
{
  "dependencies": {
    "picus": "2.6.12",
    "@picus/player": "2.6.12",
    "@picus/gif": "2.6.12"
  }
}
```

This is especially true if the packages are installed over time and not all at once. Removing the `^` character is a good way to ensure all Picus packages have the same version.

## Packages that depend on Picus

If you make a Picus library and put it on npm, make Picus a `peerDependency` and `devDependency` instead of a dependency:

```json title="package.json"
{
  "name": "my-picus-library",
  "peerDependencies": {
    "picus": "*"
  },
  "devDependencies": {
    "picus": "^2.6.11"
  }
}
```

This will make sure that when you then install `my-picus-library` into your Picus project, that no duplicates of Picus will be introduced:

```json
{
  "dependencies": {
    // No version mismatch will be introduced because `picus`
    // is not a direct dependency of `my-picus-library`
    "picus": "2.7.0",
    "my-picus-library": "1.0.0"
  }
}
```

## Checking which versions have been installed

Just because the version in the package.json is `^2.6.11` does not mean that the version `2.6.11` got installed. Use the command:

```bash
npx picus versions
```

to see which packages and versions your project contains.
