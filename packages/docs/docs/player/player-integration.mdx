If you are using the player, a common desire is to share the code with your Picus Studio and/or server-side rendering. With the correct setup, you can write the component once and use it for previewing, displaying and rendering.

:::note
Picus and your React app use a different Webpack config. Therefore, if you want to override your [Webpack configuration](/docs/webpack), you need to override both the Picus one and the React app one.
:::

## Manual setup

Set up a React project with your preferred setup from the [Official React docs](https://react.dev/learn/start-a-new-react-project). Popular choices are:

- [Next.js](https://nextjs.org/learn/basics/create-nextjs-app/setup)
- [React Router](https://reactrouter.com/start/framework/installation)
- [Vite](https://vitejs.dev/guide/)

:::note
While you can still use [Create React App](https://create-react-app.dev), it is not being actively recommended by the React team anymore.
:::

Afterwards, create a subfolder for Picus within your project and add three files: An index file, a `Root.tsx` file for your list of compositions, and a file with your composition. It could look like this:

```diff
 └── src/
+  ├── picus/
+  │   ├── index.ts
+  │   ├── MyComp.tsx
+  │   └── Root.tsx
   └── app/
       └── App.tsx
```

Your composition (`picus/MyComp.tsx` in the example) could look for example like this:

```tsx twoslash
export const MyComp: React.FC<{text: string}> = ({text}) => {
  return <div>Hello {text}!</div>;
};
```

Your list of videos (`picus/Root.tsx` in the example) could look like this:

```tsx twoslash
// @allowUmdGlobalAccess
// @filename: ./MyComp.tsx
export const MyComp = () => <></>;

// @filename: index.tsx
// ---cut---
import {Composition} from 'picus';
import {MyComp} from './MyComp';

export const MyVideo = () => {
  return (
    <>
      <Composition component={MyComp} durationInFrames={120} width={1920} height={1080} fps={30} id="my-comp" defaultProps={{text: 'World'}} />
    </>
  );
};
```

Your index file (`picus/index.ts` in the example) could look like this:

```tsx twoslash
// @filename: ./Root.tsx
export const MyVideo: React.FC<{text: string}> = () => <></>;

// ---cut---
import {registerRoot} from 'picus';
import {MyVideo} from './Video';

registerRoot(MyVideo);
```

:::tip
Don't move these statements together into one file, as you might break hot reloading.
:::

## Using the Picus Studio

You can open the Picus Studio using the `npx picus studio` command:

```bash
npx picus studio src/picus/index.ts
```

:::note
Before v4.0, the command was `npx picus preview`.
:::

We recommend adding a new script into your package.json for easy access:

```diff
  "scripts": {
+    "picus": "picus studio src/picus/index.ts"
  }
```

## Adding `<Player />` to your app

Anywhere in your app, import the [`<Player />`](/docs/player) component and your Composition component.

```tsx twoslash
// @allowUmdGlobalAccess
// @filename: ./picus/MyComp.tsx
export const MyComp = () => <></>;

// @filename: index.tsx
// ---cut---
import {Player} from '@picus/player';
import {MyComp} from './picus/MyComp';

export const App: React.FC = () => {
  return (
    <Player
      component={MyComp}
      inputProps={{text: 'World'}}
      durationInFrames={120}
      compositionWidth={1920}
      compositionHeight={1080}
      fps={30}
      style={{
        width: 1280,
        height: 720,
      }}
      controls
    />
  );
};
```

:::note
Pass your React component directly to the `component` prop. Don't pass the list of compositions.
:::

If everything worked, you can now run your webapp and preview your video.

## Creating a bundle for server-side rendering

In any Node.JS context, you can call [`bundle()`](/docs/bundle) to bundle your video using Webpack and to server-side render the video. You need to add `@picus/bundler` to your package.json for this.

```ts twoslash title="server.tsx"
import path from 'path';
import {bundle} from '@picus/bundler';

const bundled = await bundle(path.join(process.cwd(), 'src', 'picus', 'index.ts'));
```

See [Server-side rendering](/docs/ssr) for a full example.

:::tip
When using Lambda, you don't need this, you can use the CLI or [`deploySite()`](/docs/lambda/deploysite) which will bundle the video for you.
:::
