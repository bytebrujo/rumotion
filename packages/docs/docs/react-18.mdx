## Updating packages

To use React 18's newest features, you need at least version `3.0.0` of Picus.

```diff
- "picus": "2.6.15"
- "@picus/bundler": "2.6.15"
- "@picus/cli": "2.6.15"
- "@picus/renderer": "2.6.15"
+ "picus": "3.0.0"
+ "@picus/bundler": "3.0.0"
+ "@picus/cli": "3.0.0"
+ "@picus/renderer": "3.0.0"
```

You need to upgrade both `react` and `react-dom`:

```diff
- "react": "17.0.1"
- "react-dom": "17.0.1"
+ "react": "18.2.0"
+ "react-dom": "18.2.0"
```

If you use TypeScript, update to the newest types as well:

```diff
- "@types/react": "17.0.3"
+ "@types/react": "18.0.0"
```

Run `npm i`, or `yarn`, or `pnpm i` afterwards, matching your package manager.

## Fixing `React.FC` types

The types for `React.FC` have changed to no longer include `children`. If you get a type error, change

```tsx
const MyComp: React.FC = ({ children }) => <div>{children}</div>;
```

to:

```tsx
const MyComp: React.FC<{
  children: React.ReactNode;
}> = ({ children }) => <div>{children}</div>;
```

## No action required for `createRoot()`

If React 18 is installed, Picus will automatically use `createRoot()` from `react-dom/client` instead of `render` from `react-dom`. You don't need to do anything.
