You can deploy the Picus Studio as a static site, for example to Vercel or Netlify.  
Server-side rendering will not be available, but if you enable [client-side rendering](/docs/client-side-rendering), you can render videos directly in the browser.  
The deployed URL may also be used as a [Serve URL](/docs/terminology/serve-url) to pass to rendering APIs.  
Make sure you are on at least v4.0.97 to use this feature - use `npx picus upgrade` to upgrade.

## Export the Picus Studio as a static site

To export the Picus Studio as a static site, run the [`picus bundle`](/docs/cli/bundle) command:

```bash
npx picus bundle
```

The output will be in the `build` folder.  
We recommend to add `build` to your `.gitignore` file. The command will offer to do it for you when you run it.

## Deploy to Vercel

To deploy to Vercel, connect your repository to Vercel.

In the "Build and Output" settings, enable "OVERRIDE" and set the following:

- **Build Command**: `bunx picus bundle`
- **Output Directory**: `build`
- **Installation Command**: _Leave default_

![Alt text](https://github.com/picus-dev/picus/assets/86873911/ac38e1fb-0c95-4f64-b2b9-b72b90d69f22)

:::note
Using bunx as a script runner is slightly faster than using `npx`.
:::

## Deploy to Netlify

Connect your repository to Netlify.

- **Base directory**: _Leave default_
- **Build command**: `npx picus bundle`
- **Publish directory**: `build`
- **Functions directory**: _Leave default_

## Deploy to GitHub Pages

<Step>1</Step> Create the following file in your repo:

```yaml title=".github/workflows/deploy-studio.yml"
name: Deploy Picus studio
on:
  workflow_dispatch:
  push:
    branches:
      - 'main'
permissions:
  contents: write
jobs:
  render:
    name: Render video
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@main
      - name: install packages
        run: npm i
      - name: Bundle Studio
        run: npx picus bundle --public-path="./"
      - name: Deploy Studio
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          folder: build
```

The above code will deploy the bundled Picus Studio to a branch `gh-pages`.

:::note
The --public-path flag for `npx picus bundle` is available only from picus version `4.0.127`
:::

<Step>3</Step> Go to the settings of your repository (github.com/[username]/[repo]/settings/pages) <br />

<Step>4</Step> In the Branch section, select the <code>gh-pages</code> branch and click save.

## Rendering from a URL

The deployed URL is a [Serve URL](/docs/terminology/serve-url) can also be used to render a video:

If you open a deployed Studio in read-only mode, you can open the Render modal and click **Copy command** to get the CLI command for the current settings.

<TabItem value="ssr">

```tsx twoslash title="render.mjs"
const outputLocation = '/path/to/frames';

import {renderMedia, selectComposition} from '@picus/renderer';

// ---cut---
const inputProps = {
  titleText: 'Hello World',
};

const serveUrl = 'https://picus-helloworld.vercel.app';

const composition = await selectComposition({
  serveUrl,
  id: 'HelloWorld',
  inputProps,
});

await renderMedia({
  composition,
  serveUrl,
  codec: 'h264',
  inputProps,
});
```

Refer to [`renderMedia()`](/docs/renderer/render-media) to see all available options.
