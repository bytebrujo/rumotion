We now make it possible to export the Picus Studio as a static website and deploy it to any hosting provider.

You can share the resulting URLs with your team, clients or friends. They can preview the video, and change the parameters to customize the video.

You can also pass the URL to any Picus rendering API, such as [`npx picus render`](/docs/cli/render) and [`renderMedia()`](/docs/renderer/render-media), without having to clone the code.

## Deploy as a static website

Use [`npx picus bundle`](/docs/cli/bundle) to export the Picus Studio as a static website. Enter this build command on Vercel, Netlify or another provider to continuously deploy the Studio.

## Render from a URL

You may use the URL to render:

```bash
npx picus render https://watercolor-map.vercel.app
```

You can use [`--props`](/docs/cli/render#--props) to parameterize the video.

All render APIs support this: [`renderMedia()`](/docs/renderer/render-media), [`renderStill()`](/docs/renderer/render-still), [`getCompositions()`](/docs/renderer/get-compositions), [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda), [`renderFrames()`](/docs/renderer/render-frames), [`renderStillOnLambda()`](/docs/lambda/renderstillonlambda), [`getCompositionsOnLambda()`](/docs/lambda/getcompositionsonlambda), [`renderMediaOnCloudRun()`](/docs/cloudrun/rendermediaoncloudrun), [`renderStillOnCloudRun()`](/docs/cloudrun/renderstilloncloudrun), [`npx picus render`](/docs/cli/render), [`npx picus still`](/docs/cli/still), [`npx picus compositions`](/docs/cli/compositions), [`npx picus lambda render`](/docs/lambda/cli/render), [`npx picus lambda still`](/docs/lambda/cli/still), [`npx picus lambda compositions`](/docs/lambda/cli/compositions), [`npx picus cloudrun render`](/docs/cloudrun/cli/render) and [`npx picus cloudrun still`](/docs/cloudrun/cli/still).

## Deploy the Studio to a server

You may also deploy the Picus Studio as a long-running server, for example on Fly.io or Render.com.  
The advantage is that the Render Button stays active, meaning you can render and download videos!

[Read here how to deploy the Picus Studio to a server](/docs/studio/deploy-server).
